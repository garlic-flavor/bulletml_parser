インスタンスをそれぞれの機能を表現したインターフェイスへと実行時キャスト することで利用する。
サイズや現在位置を示すのに、byteではなく「要素数」を使うので注意。

現在位置を保持している。

const ulong tell();

現在位置。単位は要素量。

interface TIReadSeq(T): IAIO, ITell!T;

buf を値で埋める。実際に埋められた分をスライスで返す。

現在位置から s 要素分進める。

バイト数ではない。

バイト数ではない。

void seekSet(long s);
void seekEnd(long s);

s の単位は要素数

無限直線 arr が、TRIANGLE を形成する face、 vertex と交差するか、 どう交差するか？(表から？裏から？)を判定している。
点 arr.p が TRIANGLE表面上にあり、直線 arr.v がポリゴン平面と平行の場合、 直線は「TRIANGLE表から」「交差」と判定される。
点 arr.p が TRIANGLE表面上になく、直線 arr.v が TRIANGLE表面と平行に TRIANGLE表面上を通る場合、直線は「交差なし」と判定される。
それ以外の場合、TRIANGLE表面、境界および頂点上は「交差」と判定される。

Params:

const(Arrow!PRECISION) arr 調べたい直線。arr.v は正規化されているものとする。
正規化されていない場合、表裏判定は同じだが hs.dist の値は デタラメになる。
uint[] face TRIANGLEを定義するインデックス配列
Vector3!PRECISION[] vertex TRIANGLEを成形する頂点配列

Throws:
face.length < 3 もしくは face が示す先より vertex が短かかった場合、 Range violation が投げられる。

メッシュは閉じており、(OpenGL的に)表裏が正しいと仮定する。
(OpenGL では反時計回り順のポリゴンが表)
任意方向での最寄りのポリゴンの裏が見えているかどうかを判定している。
裏が見えていれば、点はメッシュの内にある。
